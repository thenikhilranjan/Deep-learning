## 3) Analysis of an NLP-related Project: FinGPT (Approx. 260-300 words)

### 3.1 Project Overview
- Project name + link: `FinGPT` ([GitHub](https://github.com/AI4Finance-Foundation/FinGPT))
- Academic reference: `Yang, Liu, & Wang (2023)` ([arXiv](https://arxiv.org/abs/2306.06031))
- Project type: Open-source financial NLP/LLM framework with data engineering + model adaptation + application layers.

### 3.2 Problem the Project Solves
- Financial NLP systems need timely, domain-specific, high-quality data, but proprietary systems are expensive and less accessible.
- FinGPT addresses:
  - Data access and curation barriers.
  - High adaptation cost of financial-domain LLMs.
  - Need for reproducible, transparent financial NLP workflows.

### 3.3 Potential Impact on Industry and Society
- **Industry impact**:
  - Faster prototyping/deployment of financial NLP use cases (sentiment analysis, advisory support, document intelligence).
  - Lower experimentation cost through lightweight adaptation methods and open tooling.
  - Better alignment with enterprise workflows requiring data pipelines and continuous updates.
- **Societal impact and risks**:
  - Positive: democratizes access to advanced finance-focused NLP capabilities.
  - Risks: model hallucination, bias amplification, and over-reliance in high-stakes decisions.
  - Mitigation discussion: human oversight, validation pipelines, guardrails, and governance standards.
